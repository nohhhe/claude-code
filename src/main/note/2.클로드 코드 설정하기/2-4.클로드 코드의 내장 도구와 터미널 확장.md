2-4.클로드 코드의 내장 도구와 터미널 확장
=========================
## ⭕ 클로드 코드의 내장 도구 이해하기
클로드는 파일 시스템 조작, 터미널 명령어 실행, 웹 리소스 접근 등 개발에 필요한 대부분의 작업을 직접 수행할 수 있는 강력한 내장 도구들을 제공합니다.

## ⭕ 파일 시스템 도구
- Read: 파일 읽기
- Write/Edit: 파일 생성 및 수정
- Glob/Grep: 파일 검색
  - Glob: 파일명/패턴으로 검색
  - Grep: 파일 내용으로 검색
- LS: 디렉터리 구조 탐색

## ⭕ 시스템 실행 도구
- Bash: 터미널 명령어 실행

## ⭕ 웹 리소스 도구
- WebFetch: 웹 콘텐츠 가져오기
- WebSearch: 웹 검색 수행

## ⭕ 작업 관리 도구
- TodoWrite: 작업 계획 및 추적

## ⭕ 내장 도구 조합 패턴
### ◉ 코드 품질 향상 워크플로
- 1단계: 프로젝트 전체 구조 파악
  - prompt: 프로젝트 구조를 분석하고 코드 품질 개선 계획을 수립해줘
  - LS로 구조 파악 -> Glob으로 파일 검색 -> TodoWrite로 계획 수립
- 2단계: 코드 분석 및 문제점 발견
  - prompt: 타입스크립트 오류와 린트 경고를 모두 찾아서 수정해줘
  - Bash tsc, eslint 실행 -> Grep으로 에러 패턴 검색 -> Edit로 수정
- 3단계: 테스트 및 검증
  - prompt: 수정된 코드를 테스트하고 커버리지를 확인해줘
  - Bash로 테스트 실행 -> Read로 결과 분석 -> TodoWrite로 추가 작업 계획

### ◉ 신규 기능 개발 워크플로
- 1단계: 요구사항 분석 및 기술 조사
  - prompt: 사용자 인증 기능을 개발하기 위한 기술 조사해줘
  - WebSearch로 최신 기술 조사 -> WebFetch로 문서 참조 -> TodoWrite로 계획 수립
- 2단계: 코드 구현
  - prompt: JWT 기반 인증 시스템을 구현해줘
  - Write로 새 파일 생성 -> Edit로 기존 파일 수정 -> Read로 코드 검토
- 3단계: 통합 및 테스트
  - prompt: 인증 기능을 기존 시스템에 통합하고 테스트해줘
  - Grep으로 관련 코드 검색 -> Edit로 통합 -> Bash로 테스트 실행

## ⭕ 터미널 도구 활용
클로드 코드의 가장 강력한 기능 중 하나는 터미널 도구와의 직접 연동입니다. 깃, npm/yarn, 빌드/배포 도구 등 실제 개발 워크플로를 자동화할 수 있습니다.

### ◉ 깃 워크플로 자동화
클로드 코드는 깃과 직접 상호작용하며 브랜치 생성, 커밋 메시지 작성, 병합 처리까지 자동화합니다.

### ◉ npm/yarn 패키지 관리
클로드는 npm 또는 yarn을 활용해 의존성 관리와 보안 점검을 자동화합니다.

### ◉ 빌드 및 배포 자동화
클로드는 빌드 최적화부터 배포 전 검증까지 자동화된 흐름을 만들어 줍니다.

### ◉ 도커 컨테이너 관리
클로드 코드는 도커와 직접 연동하여 컨테이너 상태를 진단하고, Dockerfile 최적화까지 자동으로 처리할 수 있습니다.

### ◉ 프로세스 및 시스템 관리
클로드는 시스템 레벨의 프로세스와 리소스 관리까지 지원합니다.

## ⭕ 도구 활용 베스트 프랙티스
### ◉ 효율적인 도구 사용법
- 작업 범위 명확 하기: 너무 큰 작업은 단계별로 나누어 실행
- 도구 조합 활용: 단일 도구보다는 여러 도구를 조합하여 사용
- 결과 검증: 자동화된 작업도 항상 결과를 확인
- 점진적 자동화: 간단한 작업부터 시작해서 점차 복잡한 작업으로 확대

### ◉ 주의사항
- 권한 관리: 민감한 작업은 항상 확인 후 실행
- 백업 우선: 중요한 변경 전에는 항상 백업
- 로컬 테스트: 프로덕션 환경 작업 전 로컬에서 충분히 테스트
